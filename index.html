Shader Analysis

The sketch lines are the noise layer.

Most 3D scenes compute lighting per-fragment. This extraction found the opposite: every surface is MeshBasicMaterial — no diffuse, no specular, no environmental light anywhere in the scene.

The "noise layer" in Phase 3 isn't a noise function. It's gestural pen marks drawn as SVG paths at z-index 2 above the WebGL canvas. Each mark has a slight positional drift from the geometry underneath — the natural registration error of drawing over a reference.

The color field is purely position-driven. The teal bar counter is the sole high-saturation element in an otherwise 0.45-saturation world — the chromatic anchor that makes everything else recede.

Phase 1Single-point perspective. Vanishing point upper-left. Hard clip edges. No distortion. Zero symmetry.
Phase 2Rectangles dominant. Shelf slabs, counter slab, chair armature shells. Book spine arrays. Hard union only.
Phase 3Zero noise on fills. Sketch lines = gestural SVG pass above canvas. Darken multiply ~55%. Draws in via strokeDashoffset on load.
Phase 5Hard composite fills. Sketch SVG: multiply layer. Shadow: flat mauve PlaneGeometry 0.01 units above floor.
Phase 6UNLIT. MeshBasicMaterial everywhere. No lights. ACESFilmic tone mapping for warmth only. Shadow = geometry.
Phase 8SVG paths draw in on load (GSAP 16ms stagger). Camera drifts 0.1Hz ±0.06. Scroll proxy z=6.5→15.

8-Phase Extraction

Phase 1 — Coordinate Space

Single-Point Perspective

Cartesian · Vanishing upper-left · No distortion

Vanishing point toward the door region. Cartesian architectural grid. Hard clip at all edges — no vignette, no fade. Zero distortion functions. Zero symmetry operators. Camera placed inside the room at table height.

Phase 2 — Distance Fields

Rectangle Arrays · Shell Modifier

BoxGeometry dominant · Hard union only

Dominant shape language: boxes. Shelf slabs (thin, horizontal), counter slab (thick), book spine arrays (thin column, repeated). Chairs use a shell pattern — wireframe armature with flat cushion fills. Hard union operations only, no smooth blending anywhere.

Phase 3 — Noise Profile

Zero Noise · SVG Marks As Overlay

No noise functions · Gestural SVG · strokeDashoffset

No noise functions on any fill surface. The perceived texture is entirely the SVG sketch layer: pen marks on a fixed z-index layer above the WebGL canvas. Each path animates from invisible to visible via strokeDashoffset on load. 16ms stagger per mark.

Phase 4 — Color Field

Discrete Position-Driven Fills

8 hex values · One color per region · No mixing

Eight hex values, each assigned to exactly one spatial region. No gradient between surfaces. No color mixing. The teal bar counter runs at full saturation while everything else sits at approximately 0.45 — one chromatic focal point that pulls the eye.

Phases 5 + 6 — Blend + Lighting

Unlit · Shadow as Geometry

MeshBasicMaterial · No lights · Mauve polygon

MeshBasicMaterial on every single surface. No lights added to the Three.js scene. ACESFilmic tone mapping for warmth only. The floor shadow is a flat mauve PlaneGeometry sitting 0.01 units above the floor. Sketch SVG: CSS multiply blend mode.

Phase 8 — Temporal

0.1Hz Drift · Draw-In Animation

Sine drift · SVG sequential reveal · Scroll proxy

Two animations exist. On load: all SVG sketch paths draw in sequentially via GSAP, 16ms stagger. Camera: continuous sine drift at 0.1Hz amplitude ±0.06 units — imperceptible motion that makes the scene feel alive. Scroll drives the proxy from z=6.5 through z=15.
